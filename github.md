# GitHub 사용 방법 (초보자 가이드)

GitHub는 Git을 기반으로 하는 분산 버전 관리 시스템이자, 개발자들이 코드를 저장하고 협업할 수 있는 클라우드 플랫폼입니다.

## 1. 주요 개념
- **Repository (저장소):** 프로젝트 코드와 변경 이력이 저장되는 공간입니다.
- **Commit (커밋):** 코드의 변경 사항을 저장하는 단위입니다.
- **Push (푸시):** 로컬 컴퓨터에 저장된 커밋들을 원격 저장소(GitHub)로 업로드합니다.
- **Pull (풀):** 원격 저장소(GitHub)의 변경 사항을 로컬 컴퓨터로 다운로드하여 병합합니다.
- **Branch (브랜치):** 독립적인 작업 공간을 만들어, 메인 코드에 영향을 주지 않고 새로운 기능을 개발할 수 있게 합니다.

## 2. 기본적인 사용 흐름

### (1) 로컬 저장소 생성 및 초기화
```bash
git init
```

### (2) 변경된 파일 추가
```bash
# 특정 파일 추가
git add <파일명>

# 모든 변경된 파일 추가
git add .
```

### (3) 커밋하기
```bash
git commit -m "커밋 메시지 (예: 기능 A 추가)"
```

### (4) 원격 저장소 연결 (최초 1회)
```bash
git remote add origin <GitHub 저장소 URL>
```

### (5) 변경 사항 업로드 (Push)
```bash
git push -u origin main
```

## 3. 다른 사람의 프로젝트 가져오기 (Clone)
GitHub에 있는 프로젝트를 내 컴퓨터로 복사해옵니다.
```bash
git clone <GitHub 저장소 URL>
```

## 4. 변경 사항 동기화 (Pull)
협업 중이거나 다른 곳에서 작업한 내용을 내 로컬 저장소에 반영합니다.
```bash
git pull origin main
```

이 가이드를 통해 기본적인 GitHub 사용법을 익히고, 더 나아가 협업에 필요한 Pull Request(PR) 등 다양한 기능들을 활용해 보세요!
